AI Agent Architecture

Araca Insights — LLM-Powered Semiconductor Analytics Agent

ai/

1High-Level Architecture

The user types in a Dash chat interface. Messages route through callbacks to the AgentEngine, which maintains a conversation with the Ollama-hosted qwen3.5 model. When the LLM requests tool calls, the engine dispatches them via AgentTools to three backends: DataManager (file data access), AutoMLManager (model training and prediction), or Plotly (chart generation). Results flow back to the LLM for interpretation, and final responses stream incrementally to the UI.

2Agent Tool-Calling Loop

Each user message triggers a synchronous loop running in a background thread. The engine calls Ollama with the full message history and 13 registered tools. Ollama streams back text tokens and/or tool call requests. If tool calls are present, the engine executes each one, appends the result as a tool-role message, and loops back for another Ollama call. The loop terminates when Ollama returns a text-only response (no tool calls) or after 8 rounds. A final StreamChunk("done") signals completion.

3Streaming Pipeline

Background Thread

_process_message_sync()

Runs the Ollama conversation loop in a daemon thread so it doesn't block the Dash event loop. Generates StreamChunk objects as events occur.

Thread-Safe Queue

Queue[StreamChunk]

FIFO buffer between producer (engine thread) and consumer (Dash callback). Chunk types:

text tool_start tool_end chart prefill done error

Dash Polling Callback

every 200ms

Drains all available chunks each tick. Updates the chat UI incrementally — text accumulates into Markdown bubbles, tool indicators toggle, and charts route to canvas tabs.

The engine runs in a daemon thread to avoid blocking the Dash event loop. It pushes StreamChunk objects onto a thread-safe Queue. A Dash interval callback fires every 200ms, drains all available chunks, and routes them to the UI: text chunks accumulate into a streaming Markdown bubble, tool_start/tool_end toggle indicator pills, chart figures route to canvas tabs, and prefill payloads populate the prediction form.

4Agent Tools (13 Functions)

Data Tools

4 functions
get_dataset_summary
Overview of all loaded files, feature ranges, and categorical value counts.
get_file_details
Metrics and attributes for a single .dat polishing file.
get_feature_statistics
Descriptive stats (mean, std, min, max), optionally grouped by a categorical.
detect_outliers
IQR-based outlier detection for any numerical feature.

ML Tools

3 functions
run_automl
Train FLAML AutoML pipeline — searches 9 estimators with nested cross-validation.
open_prediction_form
Pre-fill the canvas prediction form with specific parameter values.
get_model_diagnostics
Residual stats, CV metrics, hyperparameters, and data quality warnings.

Chart Tools

6 functions
generate_scatter
Scatter plot with optional color-by, filtering, and correlation coefficient.
generate_distribution
Histogram or grouped box plot for a single feature.
generate_bar_chart
Grouped bar chart with error bars for categorical comparisons.
generate_correlation_heatmap
Correlation matrix heatmap for all or selected numerical features.
generate_time_series
In-run time-series plot for a specific .dat file's measurements.
generate_model_plots
4 diagnostic plots: predicted vs actual, feature importance, residuals, residual distribution.

All 13 tools are methods on the AgentTools class. Ollama auto-generates JSON schemas from Python type annotations and docstrings. The engine passes these schemas to Ollama at each call, and the LLM selects which tool to invoke. Chart tools return both a Plotly figure (routed to the UI) and a text summary with exact numbers (given to the LLM so it can cite real data, not guess).

5File Responsibility Map

agent.py
class AgentEngine
  • Orchestrates LLM conversation loop (up to 8 tool-calling rounds)
  • Streams responses via Queue[StreamChunk]
  • Dispatches tool calls, prunes history at ~48K chars
  • Thread-safe with threading.Lock
tools.py
class AgentTools
  • 13 tool functions (4 data, 3 ML, 6 chart)
  • Thin wrappers around DataManager and AutoMLManager
  • Returns Plotly figures + text summaries for LLM context
  • Auto-registered via Python type annotations
automl.py
class AutoMLManager
  • FLAML AutoML wrapper (9 estimators: lgbm, xgboost, rf, etc.)
  • Nested 5-fold cross-validation for honest metrics
  • Per-prediction uncertainty from tree ensemble variance
  • Category dtype snapshots for training/predict alignment
callbacks_agent.py
Dash Callbacks (8+)
  • Bridges chat UI to AgentEngine
  • 200ms polling drains Queue, updates chat incrementally
  • Routes charts to canvas tabs, prefills prediction form
  • Manages status badges (model, data, Ollama health)
ollama_manager.py
class OllamaManager
  • Starts/stops bundled or system Ollama server
  • Health checks via HTTP, model availability verification
  • Auto-detects system (port 11434) vs bundled (port 11435)
  • Graceful shutdown with 5s timeout then kill
__init__.py
Module Exports
  • Exports AgentEngine
  • Exports OllamaManager
  • Exports AutoMLManager
  • Clean public API for the rest of the app

The ai/ module follows a clean separation of concerns: the engine manages the conversation loop and streaming, tools provides the LLM-callable interface, automl encapsulates all ML logic, callbacks bridges to the Dash UI framework, and ollama_manager handles the server lifecycle.
